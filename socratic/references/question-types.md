# Socratic Question Types — Detailed Reference (소크라테스식 질문 유형 상세)

## Quick Reference Card

```
Type 1: CLARIFY      → "What do you mean?"        (무슨 뜻인가요?)
Type 2: ASSUMPTIONS  → "What are you assuming?"    (어떤 가정을 하고 있나요?)
Type 3: EVIDENCE     → "How do you know?"          (어떻게 아시나요?)
Type 4: PERSPECTIVES → "What would others say?"    (다른 사람은 뭐라 할까요?)
Type 5: IMPLICATIONS → "What follows from that?"   (그로부터 무엇이 따르나요?)
Type 6: META         → "Why ask this question?"    (왜 이 질문을 하는 건가요?)
```

---

## Type 1: Clarifying Questions (명확화 질문) — Deep Dive

### Purpose
Surface what's hidden in vague language. Most disagreements are actually definitional.

### Diagnostic Signs You Need Type 1
- Abstract nouns: "quality", "culture", "innovation", "leadership"
- Pronouns without clear referents: "they", "it", "that thing"
- Absolutes: "always", "never", "everyone", "nobody"
- Jargon or buzzwords

### Question Bank

| Context | Question |
|---------|----------|
| Vague term | "When you say 'quality', what specifically do you mean in this context?" |
| Broad claim | "Can you narrow that down? Which part specifically?" |
| Confusion | "I want to make sure I understand. Are you saying X or Y?" |
| Implicit context | "What's the specific situation you have in mind?" |
| Complexity | "Can you break that into smaller pieces?" |
| Definition dispute | "How would you define [term] for this discussion?" |

### Exercise
Take any statement and ask: "Which word in this sentence would Socrates want defined?"

> "We need better communication in our team."
> → What does "better" mean? What does "communication" include? Which "team"?

## Type 2: Probing Assumptions (가정 탐구) — Deep Dive

### Purpose
Every argument rests on invisible foundations. These questions make the invisible visible.

### Common Hidden Assumptions in Business

| Statement | Hidden Assumption |
|-----------|------------------|
| "We need to hire more people" | More people = more output |
| "The customer is always right" | Customer preferences are stable and informed |
| "We should move fast and break things" | Speed matters more than reliability |
| "Data-driven decisions are better" | The data we have is complete and unbiased |
| "Remote work reduces productivity" | Productivity is measured correctly |

### Question Bank

| Context | Question |
|---------|----------|
| Causal claim | "Are you assuming that A causes B? Could something else explain it?" |
| Universal claim | "You're assuming this applies to all cases. Can you think of an exception?" |
| Value judgment | "What values are behind that judgment? Are those the only values that matter?" |
| Prediction | "What are you assuming about the future that makes this prediction hold?" |
| Comparison | "You're comparing X to Y. Is that a fair comparison? What's different?" |

### The Ladder of Assumptions

```
Conclusion (결론)
    ↑
Inference (추론)
    ↑
Interpretation (해석)
    ↑
Selected Data (선택된 데이터)
    ↑
Observable Data (관찰 가능한 데이터)
```

Each rung adds assumptions. Type 2 questions walk DOWN the ladder.

## Type 3: Probing Evidence (증거 탐구) — Deep Dive

### Purpose
Test whether beliefs are grounded in evidence or in habit, authority, or emotion.

### Evidence Quality Hierarchy

| Level | Type | Reliability |
|-------|------|-------------|
| 1 | Controlled experiment / RCT | Highest |
| 2 | Large-scale observational data | High |
| 3 | Multiple case studies | Medium-High |
| 4 | Single case study | Medium |
| 5 | Expert opinion | Medium-Low |
| 6 | Anecdote / personal experience | Low |
| 7 | "Common sense" / "Everyone knows" | Lowest |

### Question Bank

| Context | Question |
|---------|----------|
| Assertion | "What evidence would make you confident this is true?" |
| Data claim | "Where does that number come from? How was it measured?" |
| Authority | "Is this based on evidence, or because [authority figure] said it?" |
| Correlation | "Does that prove causation, or could they just be correlated?" |
| Sample size | "How many cases is this based on? Is that enough?" |
| Falsifiability | "What evidence would DISPROVE this? Is such evidence even possible?" |
| Counter-evidence | "What does the evidence AGAINST this look like?" |

## Type 4: Questioning Perspectives (관점 질문) — Deep Dive

### Purpose
Escape the echo chamber. Steelman the opposition.

### The Steelman Technique (강철인간 기법)

The opposite of a strawman: present the STRONGEST version of the opposing argument.

> "Before you respond to the criticism, can you state their argument in a way they would agree with?"

### Question Bank

| Context | Question |
|---------|----------|
| One-sided argument | "What's the strongest argument against this?" |
| Cultural blindness | "How would someone from a completely different background see this?" |
| Stakeholder analysis | "Who benefits from this? Who loses? What would the losers say?" |
| Historical parallel | "Has this been tried before? What happened?" |
| Devil's advocate | "If I had to argue against this, what would I say?" |
| Empathy probe | "What might make a reasonable person hold the opposite view?" |

## Type 5: Probing Implications (함축 탐구) — Deep Dive

### Purpose
Follow the logic to its endpoint. Most bad decisions come from not thinking two steps ahead.

### The "And Then What?" Chain (그리고 그 다음은?)

Keep asking "And then what?" until you reach the real consequence:

> "We should cut costs by 20%."
> "And then what?" → "We'd need to reduce headcount."
> "And then what?" → "Remaining staff would be overworked."
> "And then what?" → "Quality drops. Top performers leave."
> "And then what?" → "Revenue drops more than the cost savings."

### Question Bank

| Context | Question |
|---------|----------|
| Decision | "If we do this, what happens next? And after that?" |
| Logic chain | "If that's true, what else must be true?" |
| Consistency | "If this applies here, should it also apply to [analogous case]?" |
| Scope | "If everyone did this, what would the world look like?" (Kant's universalizability test) |
| Second-order | "What are the unintended consequences?" |
| Reversibility | "Can we undo this if we're wrong? What's the cost of reversal?" |

## Type 6: Meta Questions (메타 질문) — Deep Dive

### Purpose
Step outside the conversation and examine the conversation itself.

### When to Deploy

- The discussion is circular
- People are talking past each other
- The framing might be wrong
- Energy is dropping
- The real issue might be different from the stated issue

### Question Bank

| Context | Question |
|---------|----------|
| Circular discussion | "We keep coming back to the same point. What are we missing?" |
| Wrong frame | "Are we asking the right question, or should we reframe?" |
| Hidden agenda | "What's really at stake here for each of us?" |
| Complexity | "Is this actually one question, or several questions bundled together?" |
| Priority | "Of all the questions we could ask, is this the most important one right now?" |
| Learning | "What have we learned so far in this discussion? What's shifted?" |

## Question Sequencing (질문 순서 전략)

### The Funnel Pattern (깔때기 패턴)

```
Type 1 (Clarify)     ← Wide, open
    ↓
Type 2 (Assumptions) ← Narrowing
    ↓
Type 3 (Evidence)    ← Testing
    ↓
Type 4 (Perspectives)← Broadening again
    ↓
Type 5 (Implications)← Deepening
    ↓
Type 6 (Meta)        ← Reflecting
```

### Common Sequences

| Goal | Sequence |
|------|----------|
| Explore a new idea | 1 → 2 → 3 → 5 |
| Challenge a confident claim | 1 → 3 → 4 → 2 |
| Unstick a debate | 6 → 1 → 4 → 5 |
| Decision evaluation | 1 → 2 → 5 → 4 → 3 |
| Self-reflection coaching | 1 → 2 → 6 → 4 |

## Socratic Questioning in CBT (인지행동치료에서의 소크라테스식 질문)

Cognitive Behavioral Therapy uses Socratic questions to challenge cognitive distortions:

| Distortion | Socratic Question |
|-----------|-------------------|
| Catastrophizing (파국화) | "What's the MOST likely outcome, not the worst?" |
| Mind reading (독심술) | "How do you know what they're thinking?" |
| All-or-nothing (흑백 사고) | "Is there a middle ground between those two extremes?" |
| Overgeneralization (과잉 일반화) | "Is this true in every single case? Can you think of one exception?" |
| Emotional reasoning (감정적 추론) | "Just because you feel it, does that make it true?" |
| Should statements (당위적 사고) | "Says who? Where does that rule come from?" |

## Facilitation Tips (진행 팁)

1. **Wait time**: After asking a question, count to 10 silently. Silence is productive.
2. **One question at a time**: Never ask double-barreled questions.
3. **Follow their thread**: Don't jump to your next prepared question — follow what they say.
4. **Genuine curiosity**: If you're not genuinely interested in the answer, don't ask.
5. **Comfort with "I don't know"**: This is the most productive answer. Build from there.
